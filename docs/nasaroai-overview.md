# Nasaro AI 쉬운 설명서

## 1. Nasaro AI가 하는 일

Nasaro AI는 한 화면에서 AI를 비교하고, 토론시키고, 작업 유형별 협업 순서를 짜고, 에이전트에게 웹 작업을 맡기는 서비스입니다.

핵심 기능은 네 가지입니다.

- 비교: 같은 질문을 여러 AI 모델에 동시에 보내고 답변을 나란히 봅니다.
- 토론: 하나의 주제에 대해 여러 AI가 서로 다른 관점으로 라운드별 발언을 합니다.
- 협업: 사용자가 만들고 싶은 작업을 입력하면 조사, 구조화, 제작, 검증, 재제작 순서로 어떤 도구를 써야 하는지 플랜을 만듭니다.
- 에이전트: PC 확장 프로그램이나 Android 오버레이 앱을 통해 사용자가 보고 있는 화면 위에서 임무를 입력하고 실행합니다.

## 2. 전체 구조

서비스는 크게 웹, 서버, 확장 프로그램, Android 앱으로 나뉩니다.

- `index.html`: 사용자가 보는 메인 웹앱입니다. 비교, 토론, 협업, 설치 안내, 웹 기반 에이전트 UI가 들어 있습니다.
- `main.py`: FastAPI 서버입니다. OpenRouter API 호출, 모델 비교 스트리밍, 토론 세션, 협업 알고리즘, 에이전트 API를 처리합니다.
- `custom_agent.py`: 서버 브라우저 기반 에이전트가 페이지를 분석하고 다음 행동을 결정하는 데 쓰는 보조 로직입니다.
- `extension/`: PC 브라우저 확장 프로그램입니다. 모든 사이트에 런처와 에이전트 창을 주입하고, 실제 탭 조작은 Chrome Debugger API로 실행합니다.
- `android-agent/`: Android APK 앱입니다. 메인 앱은 Nasaro AI 사이트를 WebView로 열고, 에이전트는 시스템 오버레이 창으로 띄웁니다.
- `static/`: 사용자가 다운로드하는 APK와 확장 ZIP 파일이 들어갑니다.

## 3. 비교 기능 처리 흐름

사용자가 비교 탭에서 질문을 입력하면 프론트엔드는 여러 모델 라벨에 대해 병렬로 `/compare/stream`을 호출합니다.

서버는 OpenRouter의 무료 모델 목록을 캐시하고, 각 회사 라벨에 맞는 후보 모델을 고릅니다. 모델이 실패하거나 제한에 걸리면 다른 무료 모델로 폴백합니다. 응답은 SSE 스트리밍으로 프론트에 조금씩 내려가며, 화면에는 모델별 카드가 실시간으로 채워집니다.

## 4. 토론 기능 처리 흐름

토론 탭은 세션 ID를 기준으로 서버에 토론 상태를 저장합니다.

처음 시작하면 `/debate/start`가 호출되고, 이어서 `/debate/continue`로 다음 라운드를 진행합니다. 사용자가 중간에 의견을 넣으면 다음 라운드에 반영됩니다. 한 발언이 실패하면 해당 발언만 재시도할 수 있도록 설계되어 있습니다.

## 5. 협업 기능 처리 흐름

협업 탭은 이제 단순 추천 문장이 아니라 작업 유형별 알고리즘을 반환합니다.

서버는 사용자의 작업 문장을 보고 다음 유형 중 하나로 분류합니다.

- 동영상 제작
- 문서 제작
- 앱·웹 개발
- PPT·발표자료
- 이미지·디자인
- 데이터 분석
- 마케팅·카피
- 학습·강의
- 리서치·아이디어
- 업무 자동화

각 유형은 조사, 구조화, 제작, 검증/재제작 단계가 다릅니다. 예를 들어 동영상 제작은 훅, 장면표, 자막, BGM, 저작권 검증이 중요하고, 앱 개발은 요구사항, API, 상태, 테스트, 배포 검증이 중요합니다.

프론트는 서버가 내려준 플랜을 카드로 렌더링합니다.

## 6. PC 확장 에이전트

PC 확장 프로그램은 모든 사이트에 런처를 주입합니다.

동작 방식은 이렇습니다.

- 사용자가 확장 아이콘이나 사이트의 에이전트 버튼을 누르면 `agentEnabled`가 켜집니다.
- 백그라운드 서비스 워커가 모든 열린 탭에 `content.js`를 주입합니다.
- 런처 위치, 창 위치, 창 크기, 기록 영역 높이는 `chrome.storage.local`에 저장됩니다.
- 값이 바뀌면 모든 탭에 즉시 브로드캐스트되어 새로고침 없이 따라갑니다.
- 사용자가 임무를 입력하면 background script가 현재 탭의 DOM 요소를 스캔하고 `/agent/step`에 다음 행동 판단을 요청합니다.
- 실제 클릭, 입력, 스크롤은 Chrome Debugger API로 수행됩니다.

## 7. Android 앱 에이전트

Android 앱은 두 층으로 동작합니다.

- 메인 앱: `MainActivity`의 WebView가 Nasaro AI 사이트를 엽니다.
- 에이전트 오버레이: `FloatingService`가 시스템 오버레이 권한으로 화면 위에 질문창이나 런처를 띄웁니다.

수정 후 흐름은 다음과 같습니다.

1. 앱을 열면 Nasaro AI 사이트만 나옵니다. 런처가 자동으로 뜨지 않습니다.
2. 사이트 하단의 에이전트 버튼을 누르면 Android 브릿지가 네이티브 오버레이 서비스를 엽니다.
3. 오버레이는 `agent_overlay=1` 모드로 열려서 전체 사이트가 아니라 에이전트 패널만 표시합니다.
4. 접기를 누르면 패널이 사라지고 네이티브 런처만 남습니다.
5. 런처를 누르면 다시 에이전트 패널이 열립니다.
6. 종료를 누르면 패널과 런처가 모두 사라집니다.

Android의 일반 앱 화면을 실제로 조작하려면 시스템 접근성 서비스가 필요합니다. 현재 구조는 오버레이 질문창과 서버/웹 에이전트 실행까지 지원하며, 다른 앱의 버튼을 직접 누르는 완전 자동 조작은 접근성 서비스 추가가 다음 단계입니다.

현재 Android 1.2부터는 기본 접근성 실행 엔진이 포함되어 있습니다. 접근성 권한을 켜면 다음 명령을 실제 기기에서 처리할 수 있습니다.

- 홈 화면 이동
- 뒤로가기
- 최근 앱 화면 열기
- 설치된 앱 이름으로 실행
- 현재 화면의 텍스트를 찾아 클릭
- 현재 포커스된 입력창에 텍스트 입력
- 위/아래 스크롤

처음 실행 시 접근성 권한이 꺼져 있으면 앱이 Android 접근성 설정을 열고, 사용자가 `Nasaro AI 에이전트`를 켠 뒤 다시 명령해야 합니다.

## 8. 배포 구조

Render는 GitHub 저장소의 `main` 브랜치를 보고 배포합니다.

서버 실행 명령은 다음 구조입니다.

```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

필수 환경변수는 다음입니다.

- `OPENROUTER_API_KEY`: AI 모델 호출용 키

선택 환경변수는 다음입니다.

- `BROWSERLESS_TOKEN`: 서버 브라우저 자동화용 토큰
- `BROWSERLESS_ENDPOINT`: 별도 Browserless 엔드포인트를 쓸 때 사용

## 9. 지금 보완하면 좋은 아이디어

1. 접근성 명령 이해 범위 확장
   현재는 홈, 뒤로, 앱 실행, 텍스트 클릭, 입력, 스크롤 중심입니다. 다음 단계에서는 화면 구조를 서버 AI에 보내 더 복잡한 다단계 조작을 자동 계획하게 만들 수 있습니다.

2. 에이전트 권한 상태 진단 화면  
   오버레이 권한, 접근성 권한, 인터넷 연결, 서버 상태를 한 화면에서 보여주면 앱 오류 원인을 사용자가 바로 알 수 있습니다.

3. 작업 로그 타임라인 강화  
   에이전트가 어떤 화면을 보고 어떤 판단으로 클릭했는지 타임라인으로 보여주면 신뢰도가 올라갑니다.

4. 협업 결과 저장 기능  
   협업 플랜을 저장하고 다시 이어서 실행할 수 있게 하면 실제 프로젝트 관리 도구처럼 쓸 수 있습니다.

5. 협업 단계별 실행 버튼  
   조사 실행, 구조화 실행, 제작 프롬프트 생성, 검증 체크리스트 실행처럼 각 단계마다 바로 실행 버튼을 붙이면 협업 탭이 더 실용적입니다.

6. 사용자별 서버 저장소  
   지금 로컬 저장 중심인 기록을 계정 기반 서버 저장으로 바꾸면 기기 변경 후에도 비교/토론/협업 내역을 이어갈 수 있습니다.

7. 모델 상태 대시보드  
   무료 모델의 사용 가능 여부, 429 제한, 응답 속도를 보여주면 실패 이유를 사용자가 이해하기 쉽습니다.

8. 쿼터 정책 서버화  
   프론트 문구만이 아니라 서버에서 사용자별 쿼터를 실제로 계산하고 차감해야 서비스 운영이 안정적입니다.

9. 설치 자동 진단  
   확장 설치 여부, APK 버전, 최신 ZIP/APK 버전을 서버에서 확인해 업데이트가 필요한지 알려주면 설치 문제가 줄어듭니다.

10. 에이전트 안전 중단 버튼 강화  
   모든 실행 중 임무를 즉시 취소하고 브라우저 디버거 연결을 끊는 전역 중단 버튼을 더 크게 노출하면 안전합니다.

11. 결과물별 템플릿 라이브러리  
   문서, PPT, 영상, 앱, 마케팅, 데이터 분석별 기본 템플릿을 제공하면 사용자가 더 빠르게 시작할 수 있습니다.

12. 검증 전용 AI 단계  
   제작 AI와 검증 AI를 분리하면 같은 모델이 자기 결과를 그대로 합리화하는 문제를 줄일 수 있습니다.

13. 실패 재시도 전략 개선  
   모델 제한, 네트워크 오류, 브라우저 실행 오류를 구분해 자동 재시도와 사용자 안내를 다르게 하면 체감 안정성이 좋아집니다.

14. 모바일 전용 경량 UI  
   모바일에서는 비교 카드와 토론 덱을 더 작게, 에이전트 입력은 더 크게 배치하면 조작성이 좋아집니다.

15. 배포 상태 표시  
   현재 Render 배포 SHA, 확장 버전, APK 버전을 사이트 하단에 표시하면 새 버전 반영 여부 확인이 쉬워집니다.
