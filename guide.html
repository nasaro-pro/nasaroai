Nasaro AI 구조·아이디어 가이드

앱·웹·확장·서버가 어떻게 연결되고, 무엇을 더 만들면 좋은지 정리한 문서입니다.

← Nasaro AI로 돌아가기

1. Nasaro AI가 하는 일

비교같은 질문을 여러 AI에 동시에 보냅니다.
토론주제에 대해 AI들이 라운드별로 의견을 냅니다.
협업작업 유형별 4단계 알고리즘으로 실제 진행합니다.
에이전트PC 확장 또는 Android 오버레이로 화면 위에서 임무를 실행합니다.

2. 전체 구조

  • index.html — 메인 웹앱 UI
  • main.py — FastAPI 서버 (비교/토론/협업/에이전트 API)
  • extension/ — PC Chrome 확장 (런처·에이전트 창·탭 조작)
  • android-agent/ — Android APK (WebView + 오버레이 + 접근성 조작)
  • static/ — APK/ZIP 다운로드 파일

3. 협업 처리 흐름

사용자가 작업을 입력하면 서버가 10가지 유형 중 하나로 분류합니다.

  • 동영상 제작 · 문서 제작 · 앱·웹 개발 · PPT·발표자료 · 이미지·디자인
  • 데이터 분석 · 마케팅·카피 · 학습·강의 · 리서치·아이디어 · 업무 자동화

각 유형마다 조사 → 구조화 → 제작 → 검증/재제작 알고리즘이 다릅니다. 협업 탭에서 단계 실행 버튼을 누르면 해당 단계의 체크리스트와 도구 사용법을 AI가 생성합니다.

4. PC 확장 에이전트

  1. 에이전트 ON → 모든 탭에 content.js 주입
  2. 런처/창 위치·크기는 모든 탭에 즉시 동기화
  3. 임무 입력 → DOM 스캔 → /agent/step → Chrome Debugger로 클릭/입력

Chrome 새 탭은 확장이 Google로 연결되도록 설정되어, 새 탭에서도 런처가 표시됩니다.

5. Android 앱 에이전트

  1. 앱에서 에이전트 버튼 → 오버레이 질문창
  2. 접기 → 런처 / 다시 탭 → 질문창
  3. 접근성 ON → 홈, 뒤로, 앱 실행, 클릭, 입력, 스크롤
  4. 오버레이가 다른 앱을 가리지 않도록 실행 시 패널을 잠시 숨긴 뒤 조작

6. 모바일 제한과 현실적인 선택

iOS는 Apple 정책상 다른 앱 화면을 직접 조작할 수 없습니다. PWA로 비교·토론·협업만 가능합니다.

Android는 접근성 서비스 + 오버레이로 다른 앱 조작이 가능하지만, Play Store에 올리면 접근성·오버레이 권한 사용 목적을 Google 검수에서 설명해야 합니다. APK 직접 배포는 검수는 없지만, 사용자가 출처 허용·접근성 허용을 직접 해야 합니다.

완전히 “제한 없이” 모든 앱을 AI가 조작하는 것은 OS 보안상 불가능에 가깝습니다. 현실적인 최선은 접근성(Android) / Chrome Debugger(PC) 같은 공식 API를 쓰는 것입니다.

7. 보완·확장 아이디어 (15가지)

  1. 접근성 명령을 서버 AI와 연결해 다단계 자동 조작
  2. 오버레이·접근성·서버 상태를 한 화면에서 진단
  3. 에이전트 판단·클릭 타임라인 기록
  4. 협업 플랜 저장 및 이어하기
  5. 협업 단계별 자동 실행·검증 체크리스트
  6. 계정 기반 서버 저장으로 기기 간 동기화
  7. 모델 상태·429·속도 대시보드
  8. 서버 쿼터 실제 차감·리셋
  9. 확장/APK 버전 자동 진단·업데이트 안내
  10. 전역 에이전트 긴급 중단 버튼 강화
  11. 작업 유형별 템플릿 라이브러리
  12. 제작 AI와 검증 AI 분리
  13. 오류 유형별 자동 재시도 전략
  14. 모바일 전용 경량 UI
  15. 배포 SHA·확장·APK 버전 표시
